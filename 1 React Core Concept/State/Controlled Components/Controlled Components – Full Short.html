Controlled Components – Full Short (React)

1️⃣ Definition (Notes Format)

👉 A Controlled Component is a form element (like input, textarea, select) whose value is controlled by React state.

2️⃣ Simple Explanation

👉 Tamil:

Input box-la type pannumbodhu, adhu direct DOM control illa. React state control pannum.

👉 English:

In controlled components, React state controls the form input values.

3️⃣ Key Idea

👉 React = Single source of truth

Meaning:
👉 Input value always comes from state

4️⃣ How it works?

Flow:
User types → onChange triggers → state updates → UI re-renders → input shows updated value

5️⃣ Example (Controlled Input)

import { useState } from "react"; function App() { const [name, setName] = useState(""); return ( <div> <input type="text" value={name} onChange={(e) => setName(e.target.value)} /> <h1>{name}</h1> </div> ); }

6️⃣ Explanation

👉 value={name} → input controlled by state
👉 onChange → update state
👉 state changes → UI updates

7️⃣ Without Controlled Component (Uncontrolled)

function App() { return <input type="text" />; }

👉 Here React does NOT control input
👉 DOM controls value directly

8️⃣ Controlled vs Uncontrolled

ControlledUncontrolled
React state controls inputDOM controls input
Easier validationHard to validate
Real-time updatesNo direct sync
RecommendedLess recommended

9️⃣ Why Controlled Components?

👉 We use controlled components for:

  • ✅ Form validation
  • ✅ Real-time input tracking
  • ✅ Dynamic UI updates
  • ✅ Better control over form data

🔟 Example – Form with Controlled Inputs

import { useState } from "react"; function Form() { const [email, setEmail] = useState(""); const [password, setPassword] = useState(""); function handleSubmit(e) { e.preventDefault(); console.log(email, password); } return ( <form onSubmit={handleSubmit}> <input type="email" value={email} onChange={(e) => setEmail(e.target.value)} placeholder="Email" /> <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} placeholder="Password" /> <button type="submit">Submit</button> </form> ); }

1️⃣1️⃣ Important Rule

👉 Every controlled input must have:

  • value (state)
  • onChange (update state)

  • ✅ Easy validation

1️⃣5️⃣ Disadvantages

  • ❌ Slightly complex for beginners

🎯 Interview Answer (Important)

👉 A Controlled Component in React is a form element whose value is controlled by React state. The input value is always synced with state, and changes are handled using onChange events. This gives React full control over form data, making validation and dynamic updates easier.
