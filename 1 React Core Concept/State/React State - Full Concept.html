React State – Full Concept

1️⃣ Definition (Notes Format)

👉 State is a built-in React object used to store and manage data inside a component.

👉 When state changes, React automatically re-renders (updates) the UI.

2️⃣ Simple Explanation

👉 Tamil:

State na component-oda own data (internal data).
Data change aana UI automatic-ah update aagum.

👉 English:

State is the internal data of a component that can change over time and update the UI automatically.

3️⃣ Why We Use State?

Situation:

  • 👉 User button click pannrar.
  • 👉 Count increase aaganum.
  • 👉 UI-la new value display aaganum.
  • 👉 App automatic-ah update aaganum.

Then we use State.

✅ Manage dynamic data ✅ Update UI automatically ✅ Store component data ✅ Handle user interactions

4️⃣ How to Create State?

React Hook:

import { useState } from "react";

Example:

const [count, setCount] = useState(0);

Understanding the Syntax

PartMeaning
countCurrent state value
setCountFunction to update state
useState(0)Initial value

5️⃣ State Example

import { useState } from "react";

function Counter() {
const [count, setCount] = useState(0);

return (
<>
<h1>{count}</h1>
<button onClick={() => setCount(count + 1)}>
Increase
</button>
</>
);
}

Flow

  • Initial value = 0
  • User clicks button
  • setCount(count + 1)
  • State changes
  • Component re-renders
  • UI updates automatically

6️⃣ State is Mutable

👉 State value can change.

setCount(10);
setCount(20);
setCount(30);

7️⃣ State Causes Re-render

Important Interview Point 🔥

Whenever state changes:

👉 React re-renders the component.

setCount(count + 1);

React updates the UI automatically.

8️⃣ Types of State Data

String State

const [name, setName] = useState("Rocky");

Number State

const [age, setAge] = useState(22);

Boolean State

const [isLogin, setIsLogin] = useState(false);

Array State

const [users, setUsers] = useState([]);

Object State

const [user, setUser] = useState({
name: "Rocky",
age: 22
});

9️⃣ Updating Object State

Wrong ❌
user.name = "John";
Correct ✅
setUser({
...user,
name: "John"
});

🔟 Updating Array State

Wrong ❌
users.push("Alex");
Correct ✅
setUsers([...users, "Alex"]);

1️⃣1️⃣ State is Local to Component

👉 Each component has its own state.

<Counter />
<Counter />

Each Counter maintains separate state values.

1️⃣2️⃣ State vs Normal Variable

Normal Variable:

let count = 0;
count++;

❌ UI won't update

State:

const [count, setCount] = useState(0);
setCount(count + 1);

✅ UI updates automatically

1️⃣3️⃣ State Lifting (Important Concept)

Problem:
Child component has data. Parent component also needs that data.

Solution:
👉 Move state to Parent.
👉 Pass data using Props.

This is called Lifting State Up.

1️⃣4️⃣ State Flow Concept

State lives in Component
Parent Component
↓
Props
↓
Child Component

State usually belongs to the component that owns the data.

1️⃣5️⃣ Rules of State

  • ✅ Use useState Hook
  • ✅ Always update using setter function
  • ✅ Never modify state directly
  • ✅ State updates trigger re-render
  • ✅ State is local to component

🧠 Memory Trick

👉 Props = Outside Data

🎯 Interview Answer

👉 State is a built-in React feature used to store and manage data inside a component. State is mutable, meaning it can change over time. When the state changes, React automatically re-renders the component and updates the UI. In functional components, state is managed using the useState Hook.

💡 One-Line Summary

👉 State is the internal memory of a React component that stores data and automatically updates the UI when the data changes.

1️⃣ Functional State Update (Very Important)

Sometimes state depends on the previous state value.

❌ Normal Update
setCount(count + 1);
setCount(count + 1);
Expected: 2    Actual: 1
✅ Correct
setCount(prev => prev + 1);
setCount(prev => prev + 1);
Result: 2

Interview Point
When the next state depends on the previous state, use a functional update.

2️⃣ Asynchronous State Updates

Important Interview Question 🔥

setCount(5);
console.log(count);

Output: 0   Not: 5

Why? React schedules state updates and updates later.

Interview Answer
State updates are asynchronous and may not update immediately.

3️⃣ Batching

React groups multiple state updates into one render.

setName("Rocky");
setAge(22);
setIsLogin(true);

React performs a single re-render.

Benefit
✅ Better Performance

4️⃣ Derived State

Sometimes state can be calculated.

❌ Bad
const [fullName, setFullName] = useState("");
✅ Better
const fullName = firstName + " " + lastName;

Interview Point
Do not store state that can be derived from existing state.

5️⃣ Controlled Components

Very Important for Forms.

const [name, setName] = useState("");

<input
value={name}
onChange={(e) => setName(e.target.value)}
>

Interview Point
An input controlled by React State is called a Controlled Component.

6️⃣ Uncontrolled Components

Using Ref instead of State.

const inputRef = useRef();

React doesn't manage the value.

Interview Question
Controlled vs Uncontrolled Components.

7️⃣ Lazy Initial State

When initial value is expensive.

❌
const [data, setData] = useState(expensiveFunction());
✅
const [data, setData] = useState(() => expensiveFunction());

Runs only once.

8️⃣ Multiple States vs Single State

Multiple States

const [name, setName] = useState("");
const [age, setAge] = useState(22);

Single State Object

const [user, setUser] = useState({
name: "",
age: 22
});

Interview Point
Choose based on related data.

9️⃣ State Persistence

Important Concept

const [count, setCount] = useState(0);

When component unmounts:
👉 State is lost.

To persist: LocalStorage, SessionStorage, Database, Context API.

🔟 Key Prop & State Reset

Interview Favorite 🔥

<Counter key={id} />

When key changes:
👉 Component remounts
👉 State resets

1️⃣1️⃣ Immutable Updates

Important Rule
State should be treated as Immutable.

❌
user.name = "John";
users.push("Alex");
✅
setUser({ ...user, name: "John" });
setUsers([ ...users, "Alex" ]);

1️⃣2️⃣ State Lifecycle

Create State
↓
Read State
↓
Update State
↓
Re-render
↓
Updated UI

This flow is often asked in interviews.

1️⃣3️⃣ useState Returns Two Values

const [count, setCount] = useState(0);

Returns:

  • Current State Value
  • State Setter Function

This is called Array Destructuring.

🏆 Advanced State Interview Questions

Q1. Why should we not update state directly?
👉 React may not detect the change and re-render correctly.

Q2. What happens when state changes?
👉 React re-renders the component and updates the UI.

Q3. Does changing props cause re-render?
👉 Yes.

Q4. Does changing normal variables cause re-render?
👉 No.
