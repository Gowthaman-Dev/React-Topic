Uncontrolled Components – Full Concept (React)

1️⃣ Definition (Notes Format)

👉 An Uncontrolled Component is a form element where the DOM itself manages the input value instead of React state.

2️⃣ Simple Explanation

👉 Tamil:

Input box value-a React state control panna maata.
DOM (browser itself) manage pannum.

👉 English:

In uncontrolled components, the form data is handled directly by the DOM, not by React state.

3️⃣ Key Idea

👉 DOM = single source of truth (React illa)

4️⃣ How it works?

User types → DOM stores value → React doesn't control → we read value only when needed

5️⃣ Example (Uncontrolled Input)

👉 useRef use pannuvom

import { useRef } from "react"; function App() { const nameRef = useRef(); function handleSubmit() { alert(nameRef.current.value); } return ( <div> <input type="text" ref={nameRef} /> <button onClick={handleSubmit}> Submit </button> </div> ); }

6️⃣ Explanation

👉 ref → DOM element direct access
👉 nameRef.current.value → input value get pannrom
👉 React state use panna maatom

7️⃣ Controlled vs Uncontrolled

ControlledUncontrolled
React state controls inputDOM controls input
onChange requiredno onChange needed
Real-time updatesonly read when needed
More codeless code

8️⃣ When to use Uncontrolled Components?

👉 Use when:

  • Simple forms
  • Quick input reading
  • No validation needed
  • Performance small forms

9️⃣ Example – Form Submit

import { useRef } from "react"; function Login() { const emailRef = useRef(); const passwordRef = useRef(); function handleLogin(e) { e.preventDefault(); console.log(emailRef.current.value); console.log(passwordRef.current.value); } return ( <form onSubmit={handleLogin}> <input type="email" ref={emailRef} /> <input type="password" ref={passwordRef} /> <button type="submit">Login</button> </form> ); }

🔟 Real Life Analogy

👉 Uncontrolled Component = “Traditional paper form”

You write directly (DOM)
System doesn’t track every change
Only reads when submitted

1️⃣1️⃣ Advantages

  • ✅ Less code
  • ✅ Simple for basic forms
  • ✅ Faster (no re-render on every key press)
  • ✅ Easy to integrate with non-React libraries

1️⃣2️⃣ Disadvantages

  • ❌ No real-time validation
  • ❌ Hard to track input changes
  • ❌ Not recommended for complex forms
  • ❌ Less control over UI

1️⃣3️⃣ Important Rule

👉 React does NOT control input value
👉 DOM controls everything
👉 We just read value using ref

🎯 Interview Answer

👉 An Uncontrolled Component in React is a form element where the DOM manages the input value instead of React state. We use refs to access the input values when needed, usually during form submission. It is simpler but less powerful compared to controlled components.

1️⃣ defaultValue vs value (Very Important)

Controlled

<input value={name} onChange={(e) => setName(e.target.value) } />

Uncontrolled

<input defaultValue="Rocky" />

Difference

valuedefaultValue
Controlled ComponentUncontrolled Component
React controls valueDOM controls value
Updates with stateInitial value only

Interview Point
👉 Uncontrolled Components usually use defaultValue.

2️⃣ File Input (Most Important Interview Question)

<input type="file" />

Why Uncontrolled?
Browser does not allow React to fully control file input value.

Example:

const fileRef = useRef(); <input type="file" ref={fileRef} />

Interview Answer
File inputs are naturally uncontrolled components.

3️⃣ Uncontrolled Component Flow

User Types ↓ DOM Stores Value ↓ React Does Nothing ↓ useRef Reads Value ↓ Submit

👉 Simple:
DOM stores data until React requests it.

4️⃣ Accessing Value Without Re-render

Controlled
setName("John");
🔄 Re-render occurs

Uncontrolled
nameRef.current.value
❌ No re-render

Benefit
Better performance for very simple forms.

5️⃣ useRef Connection

Interview Question
Why do we use useRef?

👉 To access DOM elements directly without causing re-renders.

Example:

const inputRef = useRef(); inputRef.current.focus();

6️⃣ Third-Party Library Integration

Very Common Real-World Use Case

Examples:

  • jQuery plugins
  • Google Maps
  • Stripe Elements
  • Legacy JavaScript Forms

Why?

These libraries manage DOM themselves.
Uncontrolled components work better.

7️⃣ defaultChecked for Checkbox

Controlled

<input type="checkbox" checked={isChecked} />

Uncontrolled

<input type="checkbox" defaultChecked />

Interview Point
Use defaultChecked for uncontrolled checkboxes.

8️⃣ When NOT to Use Uncontrolled Components

Avoid for:

  • ❌ Complex Forms
  • ❌ Live Validation
  • ❌ Search Filtering
  • ❌ Dynamic UI Updates
  • ❌ Multi-Step Forms

Use Controlled Components Instead

9️⃣ Common Mistake

❌ Mixing Controlled & Uncontrolled

<input value={name} defaultValue="John" />

React warning.

Rule:
Choose one:
Controlled or Uncontrolled.
Never both.
