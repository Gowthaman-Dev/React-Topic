Conditional Rendering – FULL CONCEPT (React)

1️⃣ Definition

👉 Conditional Rendering is a technique in React used to display different UI elements based on a condition.

2️⃣ Simple Explanation

👉 Tamil:

Condition true-aa irundha oru UI kaamikkanum, false-aa irundha vera UI kaamikkanum. Athukku Conditional Rendering use pannuvom.

👉 English:

Conditional Rendering allows React to show different UI based on a condition.

3️⃣ Why We Use Conditional Rendering?

Real-world examples:

  • User login aagirukara?
  • Admin user-aa?
  • Data load aagiducha?
  • Products irukka?
  • Error vandhirukka?

👉 Indha situations-la UI change aaganum.

4️⃣ Basic Example

function App() { const isLoggedIn = true; return ( <div> {isLoggedIn ? <h1>Welcome User</h1> : <h1>Please Login</h1>} </div> ); }

Output:

If true → Welcome User

If false → Please Login

5️⃣ Methods of Conditional Rendering

React-la common-aa 5 methods use pannuvanga.

🟢 1. if / else

function App() { const isLoggedIn = true; if (isLoggedIn) { return <h1>Welcome</h1>; } else { return <h1>Login</h1>; } }

Use:

👉 Large conditions irundha

🔵 2. Ternary Operator (Most Used 🔥)

Syntax:

condition ? trueValue : falseValue

Example:

function App() { const isLoggedIn = true; return ( <> {isLoggedIn ? <h1>Welcome</h1> : <h1>Login</h1>} </> ); }

Why use?

✔ Short code
✔ Most common in projects

🟣 3. Logical AND (&&)

👉 Condition true-aa irundha mattum show pannum.

function App() { const isLoggedIn = true; return ( <> {isLoggedIn && <h1>Welcome</h1>} </> ); }

Output:

True → Welcome

False → Nothing

🟠 4. Logical OR (||)

👉 Default value show panna use pannuvanga.

function App() { const userName = ""; return ( <h1>{userName || "Guest"}</h1> ); }

Output:

Guest

🔴 5. Switch Statement

function App() { const role = "admin"; switch (role) { case "admin": return <h1>Admin Panel</h1>; case "user": return <h1>User Panel</h1>; default: return <h1>Guest</h1>; } }

Use:

👉 Multiple conditions irundha

6️⃣ Real Project Examples

Login Example

{ isLoggedIn ? <Dashboard /> : <Login /> }

Loading Example

{ loading ? <h1>Loading...</h1> : <h1>Data Loaded</h1> }

Admin Example

{ isAdmin && <AdminPanel /> }

7️⃣ Nested Conditional Rendering

{ isLoggedIn ? isAdmin ? <AdminPanel /> : <UserPanel /> : <Login /> }

⚠️ Avoid Too Much Nesting

Bad:

condition1 ? condition2 ? condition3 ? value1 : value2 : value3 : value4

👉 Hard to read

8️⃣ Best Practices

  • ✅ Use ternary for simple conditions
  • ✅ Use && for show/hide UI
  • ✅ Use switch for multiple cases
  • ✅ Keep conditions readable
  • ❌ Avoid deeply nested ternary operators

9️⃣ Conditional Rendering vs CSS Hide

Conditional Rendering

{ show && <Card /> }

👉 Component render aagum / aagaadhu

CSS Hide

<Card style={{display:"none"}} />

👉 Component render aagum but hidden

6️⃣ Short-Circuit Issue with &&

Important Interview Question 🔥

const count = 0; return ( <> {count && <h1>Hello</h1>} </> );

Output:

0

Why?

0 is rendered.

Better

{ count > 0 && <h1>Hello</h1> }
