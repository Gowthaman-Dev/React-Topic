React Interaction

Event Handling

Definition
Event Handling in React is used to handle user actions like click, input, submit, etc.
👉 Simple
User action handle pannrathu.
Why we use Event Handling?
We use event handling to make the UI interactive and respond to user actions.
👉 Simple
User actions-ku response kudukka.
How it works?
In React, we attach event handlers like onClick, onChange, onSubmit to elements.
👉 Simple
Button click, input change handle pannuvom.
Example events
  • onClick → button click
  • onChange → input typing
  • onSubmit → form submit
👉 Simple
Different actions for different events.
Purpose
The main purpose is to capture user actions and perform logic.
👉 Simple
User action → code run aagum.
Real meaning
Event handling connects user actions with functions in React.
👉 Simple
User action and function connect pannrathu.
🔥 Final memory line
Event Handling = React reacts to user actions like click and input.
Missing Concepts
🔹 Event Object
React automatically passes an Event Object to event handlers.
👉 Simple
User action details store aagum.

Example:

const handleClick = (event) => {
  console.log(event);
};
🔹 React Uses CamelCase
React event names use camelCase.
👉 Simple
onclick illa, onClick use pannanum.
onClick onChange onSubmit onMouseOver
🔹 Function Reference
In React, we pass a function reference, not a function call.
👉 Simple
Function-a execute panna koodathu.
✅ Correct 
<button onClick={handleClick}>
❌ Wrong 
<button onClick={handleClick()}>
🔹 Prevent Default
For forms, React uses event.preventDefault() to stop page refresh.
👉 Simple
Form submit pannumbothu page refresh stop pannum.
