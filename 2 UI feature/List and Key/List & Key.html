LIST & KEY – FULL CONCEPT (React)

1️⃣ LIST (Rendering Lists)

🔹 Definition

👉 List rendering in React means displaying multiple items from an array using JavaScript methods like map().

🔹 Simple Explanation

👉 Tamil:

Array-la irukkura multiple data-ya UI-la display panna list rendering use pannuvanga.

👉 English:

List rendering is used to display multiple items from an array in the UI.

2️⃣ Why List Rendering?

👉 Because real apps-la data always multiple items-aa irukkum:

  • Users list
  • Products list
  • Posts list
  • Comments list

3️⃣ How List Rendering works?

👉 map() function use pannuvanga

function App() { const names = ["A", "B", "C"]; return ( <ul> {names.map((name) => ( <li>{name}</li> ))} </ul> ); }

🔥 Output:

A
B
C

4️⃣ KEY Concept (VERY IMPORTANT 🔥)

🔹 Definition

👉 Key is a special attribute used in React lists to uniquely identify each item.

🔹 Simple Explanation

👉 Tamil:

List-la irukkura each item-ku oru unique ID kudukkanum. athu thaan key.

👉 English:

Key helps React identify which items are changed, added, or removed.

5️⃣ Why Key is important?

👉 React uses Virtual DOM

✔ Without key → React confused
✔ With key → React updates only changed items

6️⃣ Example WITHOUT key (Wrong ❌)

names.map((name) => <li>{name}</li>);

👉 Warning varum
👉 Performance issue

7️⃣ Example WITH key (Correct ✅)

function App() { const names = ["A", "B", "C"]; return ( <ul> {names.map((name) => ( <li key={name}>{name}</li> ))} </ul> ); }

8️⃣ Best Practice for Key

👉 Always use:

  • ✔ Unique ID
  • ✔ Database ID
  • ✔ Index (only last option)

🔥 Example with index:

names.map((name, index) => ( <li key={index}>{name}</li> ));

⚠️ But index use panna avoid pannunga if data change aagum

9️⃣ Real Life Example

👉 Instagram posts:

PostKey
Post 1id: 101
Post 2id: 102
Post 3id: 103

👉 React uses key to track each post

🔟 How React uses Key internally?

👉 React compares Virtual DOM:

  • Same key → update only that item
  • New key → add item
  • Missing key → remove item

🧠 Memory Trick

👉 List = Display multiple items
👉 Key = Identify each item uniquely

🚀 Interview Answer (PERFECT)

👉 “List rendering in React is used to display multiple items from an array using the map() function. Each item in the list must have a unique key attribute. The key helps React identify which items have changed, added, or removed, improving performance and efficient re-rendering.”

💡 One-line Summary

👉 List renders multiple items, Key uniquely identifies each item in the list.

Missing Concept

1️⃣ map() Returns a New Array

Definition
map() creates a new array by transforming each item.

const numbers = [1, 2, 3]; const result = numbers.map( (num) => num * 2 ); Output: [2, 4, 6]

Interview Point
👉 React uses map() because it returns a new array of JSX elements.

2️⃣ Rendering Objects in Lists

Real projects mostly use objects.

const users = [ { id: 1, name: "John" }, { id: 2, name: "Alex" } ]; return ( <ul> {users.map((user) => ( <li key={user.id}> {user.name} </li> ))} </ul> );

Important
✅ Most common interview example.

3️⃣ Key Should Be Unique Among Siblings

Correct
<li key={user.id}>

Rule:
Keys only need to be unique within the same list.
Not globally unique in the whole application.

4️⃣ Why Index is Not Recommended?

❌
key={index}

Problem:

A B C → Delete A → B C
Indexes change.
React may update the wrong item.

Interview Answer
Index should be used only when the list is static and never changes.

5️⃣ Key is NOT Passed as Props

Very Common Interview Question 🔥

<User key={user.id} user={user} />

Inside component:

props.key

❌ Undefined

Why?
React uses key internally.
It is not available as a normal prop.

6️⃣ Lists with Components

function User({ user }) { return <li>{user.name}</li>; } function App() { return ( <ul> {users.map((user) => ( <User key={user.id} user={user} /> ))} </ul> ); }

Interview Point
Key should be added where the list is created.

7️⃣ Filtering + List Rendering

Very Common in Real Projects

const activeUsers = users.filter( (user) => user.active ); return ( <> {activeUsers.map((user) => ( <h1 key={user.id}> {user.name} </h1> ))} </> );

Flow:
Array → filter() → map() → UI

8️⃣ List Rendering with Fragments

users.map((user) => ( <React.Fragment key={user.id}> <h2>{user.name}</h2> <p>{user.email}</p> </React.Fragment> ));

Why?
One item returns multiple elements.

9️⃣ React Rendering Flow

Array Data ↓ map() ↓ JSX Elements ↓ Virtual DOM ↓ Real DOM ↓ UI Display

Important Connection
List rendering works through Virtual DOM.

🔟 Key & Reconciliation

Very Important 🔥

Without Key:
Old List: A B C
New List: B C D
React may compare incorrectly.

With Key:
A(1) B(2) C(3) → B(2) C(3) D(4)
React knows: ❌ Remove A, ✅ Keep B, ✅ Keep C, ✅ Add D

Interview Answer
Keys help React's Reconciliation process identify changes efficiently.

1️⃣1️⃣ Common Mistakes

Missing Key

users.map((user) => ( <li>{user.name}</li> ));

❌ Warning

Duplicate Key

<li key="1">A</li> <li key="1">B</li>

❌ Wrong

Keys must be unique.

🎯 Important Interview Questions

Q1. Why use map() in React?
👉 To render multiple items from an array.

Q2. Why is key required?
👉 To uniquely identify list items.

Q3. Can we use index as key?
👉 Yes, but only for static lists.

Q4. Is key available in props?
👉 No.

Q5. How does key improve performance?
👉 Helps React identify changed, added, or removed items during reconciliation.
