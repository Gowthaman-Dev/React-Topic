useState FULL CONCEPT (React)

1️⃣ Definition

👉 useState is a React Hook used to create and manage state in functional components.

2️⃣ Simple Explanation

👉 Tamil:

Component-kulla change aagura data-va store pannavum update pannavum useState use pannuvom.

👉 English:

useState is used to store and update data inside a component.

3️⃣ Why useState?

React-la normal variable use pannina UI update aagathu.

let count = 0;

👉 count change aanaalum screen update aagathu.

React-ku state change theriyanum.

Athan useState use pannrom.

4️⃣ Syntax

const [state, setState] = useState(initialValue);

Breakdown

const [count, setCount] = useState(0);

count
👉 Current value

setCount
👉 Update function

0
👉 Initial value

5️⃣ Basic Example

import { useState } from "react"; function Counter() { const [count, setCount] = useState(0); return ( <> <h1>{count}</h1> <button onClick={() => setCount(count + 1)}> Increment </button> </> ); }

Flow

Initial value
count = 0

Click button
setCount(count + 1)

New value
count = 1

React automatically re-renders UI.

6️⃣ What is State?

👉 State = Component's internal memory

Example:

  • Counter value
  • Input field value
  • Login status
  • Theme mode
  • Cart items

7️⃣ Rules of useState

Rule 1
Only inside Functional Component

✅ Correct
function App() { const [count, setCount] = useState(0); }

Rule 2
Don't update state directly

❌ Wrong
count = 10;
✅ Correct
setCount(10);

Rule 3
State update triggers re-render

setCount(5);

👉 UI automatically updates

8️⃣ Different Types of State

String State

const [name, setName] = useState("Rocky");

Number State

const [age, setAge] = useState(21);

Boolean State

const [isLogin, setIsLogin] = useState(false);

Array State

const [users, setUsers] = useState([]);

Object State

const [user, setUser] = useState({ name: "Rocky", age: 21 });

9️⃣ Updating Object State

setUser({ ...user, age: 22 });

Why spread operator?

Without spread:

setUser({ age: 22 });

👉 name lost aagidum

🔟 Updating Array State

setUsers([...users, "John"]);

Example

Before:

["Rocky"]

After:

["Rocky", "John"]

1️⃣1️⃣ Functional Update

Best practice when using previous value.

setCount(prev => prev + 1);

Why?

React updates asynchronous-aa irukkalam.

Safe method:

setCount(prev => prev + 1);

1️⃣2️⃣ Multiple States

const [name, setName] = useState(""); const [age, setAge] = useState(0); const [city, setCity] = useState("");

1️⃣3️⃣ useState + Input

function App() { const [name, setName] = useState(""); return ( <> <input value={name} onChange={(e) => setName(e.target.value)} /> <h1>{name}</h1> </> ); }

1️⃣4️⃣ useState + Conditional Rendering

const [isLogin, setIsLogin] = useState(false); return ( <> {isLogin ? <h1>Welcome</h1> : <h1>Login</h1>} </> );

🎯 Interview Answer

👉 useState is a React Hook used to create and manage state in functional components. State represents data that can change over time. When state is updated using the setter function, React automatically re-renders the component and updates the UI.

1️⃣ useState is a Hook

Definition

👉 useState is one of React's built-in Hooks.

Interview Point
Hooks allow functional components to use React features like state and lifecycle.

import { useState } from "react";

2️⃣ State Updates are Asynchronous

Very Important Interview Question 🔥

setCount(10); console.log(count);

Output may still show old value.

Why?

React schedules state updates and updates later.

Interview Answer
👉 State updates are asynchronous and not immediate.

3️⃣ State Causes Re-render

setCount(count + 1);

Flow:

State Changes ↓ Component Re-renders ↓ UI Updates

Important

React automatically updates the UI.

4️⃣ useState Returns Two Values

const [count, setCount] = useState(0);

Returns:

Current State Value
count

Setter Function
setCount

Interview Point
This uses Array Destructuring.

5️⃣ Lazy Initial State

Important Optimization 🔥

Normal

const [data, setData] = useState(expensiveFunction());

Runs every render.

Better

const [data, setData] = useState(() => expensiveFunction());

Runs only once.

Interview Point
Called Lazy Initialization.

6️⃣ Batching

React combines multiple state updates.

setName("Rocky"); setAge(22); setCity("Chennai");

React performs:

✅ One Re-render

instead of:

❌ Three Re-renders

Benefit
Better performance.

7️⃣ State is Local to Component

<Counter /> <Counter />

Each component has its own state.

Example:

Counter 1 → count = 5

Counter 2 → count = 0

Interview Point
State is isolated per component.

8️⃣ State Persistence

const [count, setCount] = useState(0);

State exists only while component exists.

If component unmounts:

State Lost

To Persist
LocalStorage, SessionStorage, Database

9️⃣ Derived State

Avoid storing state that can be calculated.

❌
const [fullName, setFullName] = useState("");
✅
const fullName = firstName + " " + lastName;

Interview Point
Don't duplicate state unnecessarily.

🔟 State Immutability

Very Important 🔥

State should never be modified directly.

❌ Wrong
user.name = "John";
users.push("Alex");
✅ Correct
setUser({ ...user, name: "John" });
setUsers([ ...users, "Alex" ]);

Interview Answer
State should be treated as immutable.

1️⃣1️⃣ Multiple Updates Problem

❌
setCount(count + 1);
setCount(count + 1);
Expected: 2
Actual: 1
✅
setCount(prev => prev + 1);
setCount(prev => prev + 1);
Result: 2

Why?

Uses latest state value.
