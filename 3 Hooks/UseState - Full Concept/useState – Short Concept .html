Rule 1
Only inside Functional Component

✅ Correct

Rule 2
Don't update state directly

❌ Wrong

✅ Correct

2.State Causes Re-render

Flow:

State Changes ↓ Component Re-renders ↓ UI Updates

Important

4. Batching

React combines multiple state updates.

setName("Rocky"); setAge(22); setCity("Chennai");

React performs:

✅ One Re-render

instead of:

❌ Three Re-renders

Benefit
Better performance.

5.State Immutability

Very Important 🔥

State should never be modified directly.

❌ Wrong
user.name = "John";
users.push("Alex");
✅ Correct
setUser({ ...user, name: "John" });
setUsers([ ...users, "Alex" ]);

Interview Answer
State should be treated as immutable.

Important Interview Questions

Q1. What is useState?
👉 A Hook used to manage state in functional components.

Q2. Does updating state re-render component?
👉 Yes.

Q3. Can state be updated directly?
👉 No.

Q4. Why use functional updates?
👉 When next state depends on previous state.

Q5. Are state updates synchronous?
👉 No, they are asynchronous.
