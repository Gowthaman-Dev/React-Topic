useMemo – FULL CONCEPT (React)

1️⃣ Definition

👉 useMemo is a React Hook used to memoize (cache) the result of an expensive computation so that it is not recalculated on every render.

2️⃣ Simple Explanation

👉 Tamil:

Oru heavy calculation irundha, adha every render-la repeat pannaama save panni vechukradhu useMemo.

👉 English:

useMemo caches the result of a calculation and only recalculates when dependencies change.

3️⃣ Why useMemo?

👉 React re-render aana odane:

Heavy calculations repeat aagum ❌
Performance slow aagum ❌

👉 useMemo solve pannum:

✔ Avoid unnecessary calculations
✔ Improve performance
✔ Faster UI

4️⃣ Syntax

import { useMemo } from "react"; const result = useMemo(() => { return expensiveCalculation(); }, [dependency]);

5️⃣ Basic Example

import { useMemo, useState } from "react"; function App() { const [count, setCount] = useState(0); const doubleValue = useMemo(() => { console.log("Calculating..."); return count * 2; }, [count]); return ( <div> <h1>Count: {count}</h1> <h2>Double: {doubleValue}</h2> <button onClick={() => setCount(count + 1)}> Increase </button> </div> ); }

🧠 Meaning:
count change aana → recalculation
same value → cached result use

6️⃣ Without useMemo (Problem)

const result = expensiveCalculation();

👉 Every render:

function runs again ❌
performance slow ❌

7️⃣ With useMemo (Solution)

const result = useMemo(() => expensiveCalculation(), []);

👉 Only first render → calculation happens

8️⃣ When useMemo runs?

👉 Depends on dependency array:

CaseBehaviour
[]runs once
[value]runs when value changes
no dependencyruns every render

👉 useMemo is a React Hook used to memoize the result of an expensive computation. It helps optimize performance by preventing unnecessary recalculations on every render. It only recalculates the value when the dependencies change. It is mainly used for heavy computations like filtering, sorting, or complex calculations.

1️⃣ useMemo Returns a Value (Important)

Many beginners confuse useMemo and useEffect.

const result = useMemo(() => { return count * 2; }, [count]);

🧠 Remember:

useMemo → returns a value ✅
useEffect → returns nothing (or cleanup function) ✅

2️⃣ Dependency Array Importance

const total = useMemo(() => { return price * quantity; }, [price, quantity]);

If dependency changes:

✅ Recalculate

If dependency doesn't change:

✅ Use cached value

3️⃣ useMemo Execution Flow

First Render useMemo(() => { console.log("Run"); return count * 2; }, [count]); Output: Run Re-render without count change No Run count changes Run

🧠 Flow:

Render → Check Dependency → Return Cached Value OR Recalculate

5️⃣ useMemo for Object Reference

Without useMemo:

const person = { name: "Gowtham", };

Every render:

❌ New object created

With useMemo:

const person = useMemo(() => { return { name: "Gowtham", }; }, []);

✅ Same object reference reused

6️⃣ useMemo for Array Reference

const skills = useMemo(() => { return ["React", "Node", "MongoDB"]; }, []);

Useful when passing props to child components.

7️⃣ Difference: useMemo vs useCallback 🔥

Example:

const total = useMemo(() => count * 2, [count]); const handleClick = useCallback(() => { console.log("Clicked"); }, []);

9️⃣ Interview Question

Why not use useMemo everywhere?

Answer:

👉 useMemo itself has a small memory and comparison cost. Using it for simple calculations can reduce readability and may not improve performance. It should be used only for expensive computations.
