useRef – FULL CONCEPT (React)

1️⃣ Definition

👉 useRef is a React Hook that creates a mutable reference which persists across re-renders without causing a re-render when its value changes.

2️⃣ Simple Explanation

👉 Tamil:

useRef use pannitu oru value-ah store panna mudiyum. Andha value change aanaalum component re-render aagaadhu.

👉 English:

useRef lets us store a value or reference an HTML element without triggering a re-render.

3️⃣ Why We Use useRef?

👉 Mainly 2 reasons:

1. Access DOM elements directly
const inputRef = useRef();

2. Store values without re-rendering
const countRef = useRef(0);

4️⃣ Syntax

import { useRef } from "react"; const myRef = useRef(initialValue);

5️⃣ Accessing DOM Elements (Most Common Use)

import { useRef } from "react"; function App() { const inputRef = useRef(); const focusInput = () => { inputRef.current.focus(); }; return ( <> <input ref={inputRef} /> <button onClick={focusInput}> Focus Input </button> </> ); }

🧠 Explanation

👉 ref={inputRef}
React input element-ah connect pannudhu.

Actual DOM element-ah point pannum.

6️⃣ Store Value Without Re-render

import { useRef } from "react"; function App() { const countRef = useRef(0); const increase = () => { countRef.current++; console.log(countRef.current); }; return <button onClick={increase}>Increase</button>; }

Important

👉 UI update aagaadhu
👉 Re-render nadakkaadhu

  • Scroll To Element: sectionRef.current.scrollIntoView();
  • Store Previous Value: const previousValue = useRef();

  • ✅ Updating current doesn't trigger re-render

👉 useRef is a React Hook used to create a mutable reference that persists across component re-renders. It is commonly used to access DOM elements directly and store values without triggering a re-render. The value is stored inside the .current property. 1️⃣ useRef Returns an Object

Many beginners miss this.

const inputRef = useRef();

Actually returns:

{ current: undefined }

Example:

const countRef = useRef(0);

Returns:

{ current: 0 }

Interview Point
👉 useRef always returns an object with a .current property.

2️⃣ Value Persists Between Re-renders

function App() { const countRef = useRef(0); return <h1>Hello</h1>; }

Even after re-render:

countRef.current

still keeps its value.

Interview Point
👉 useRef value persists across renders.

3️⃣ Previous Value Tracking

Very Common Interview Question 🔥

import { useEffect, useRef } from "react"; function App({ count }) { const prevCount = useRef(); useEffect(() => { prevCount.current = count; }, [count]); return ( <> Current: {count} Previous: {prevCount.current} </> ); }

Use Case
Previous Count, Previous Form Value, Previous API Data

4️⃣ Store Timer IDs

Real Project Example

const timerRef = useRef(); useEffect(() => { timerRef.current = setInterval(() => { console.log("Running"); }, 1000); return () => { clearInterval( timerRef.current ); }; }, []);

Why?
Avoid losing timer ID between renders.

5️⃣ useRef with Scroll

Very Common

const sectionRef = useRef(); const scrollToSection = () => { sectionRef.current.scrollIntoView(); };

Use Cases
Go To Top, Scroll To Form, Scroll To Error

6️⃣ useRef with Focus on Mount

const inputRef = useRef(); useEffect(() => { inputRef.current.focus(); }, []);

Real Example
Login Page, Search Box, OTP Input

🔟 useRef vs createRef

createRef
Mostly used in Class Components.

const myRef = React.createRef();

useRef
Used in Functional Components.

const myRef = useRef();

Interview Point
Functional Components → useRef
Class Components → createRef

1️⃣1️⃣ Forward Ref (Advanced Concept)

Interviewers sometimes ask.

const Input = React.forwardRef( (props, ref) => { return ( <input ref={ref} /> ); } );

Why?
Pass ref from parent to child component.

1️⃣2️⃣ Callback Ref (Advanced)

Alternative to useRef.

<input ref={(element) => { console.log(element); }} />

Used rarely.

1️⃣3️⃣ Common Mistake

❌ Expecting UI Update

const countRef = useRef(0); countRef.current++; UI: <h1>{countRef.current}</h1>

May not update.
Why?
No Re-render.

Solution
Use: useState for UI data.
