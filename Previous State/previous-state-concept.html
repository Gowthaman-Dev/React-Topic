🔥 PREVIOUS STATE – Full Concepts (Class Teaching Notes)

✅ 1. Previous State na enna?

  • 👉 Old value of state

Tamil:

state update aagura munnadi iruntha value than previous state

Example:

const [count, setCount] = useState(0)

Button click panna munadi:

  • count = 0

athuku apram update panna:

  • count = 1

👉 inga 0 = previous state

✅ 2. Normal Update vs Previous State Update

  • ❌ Normal way:
setCount(count + 1)

Explain:

  • 👉 current count eduthu +1 panrom

Problem:

rapid clicks / multiple updates la wrong result varum 😬

  • ✅ Correct way (Previous State):
setCount(prev => prev + 1)

Explain slowly:

  • prev = previous count value

Tamil:

  • 👉 React namakku old value pass pannum
  • 👉 athula +1 pannrom

Board-la ezhuthunga:

  • prev = old state

✅ 3. Why previous state use pannrom?

MAIN REASON:

🔥 React state async

Tamil:

React udane update pannathu — delay irukum

So:

setCount(count + 1)
setCount(count + 1)
  • Expected: 2
  • Actual: 1 😱

But:

setCount(prev => prev + 1)
setCount(prev => prev + 1)

Result = 2 ✅

Class sentence:

👉 “Multiple updates iruntha previous state method mattum safe”

✅ 4. Functional Update Concept

Actual term:

🔹 Functional Update
setCount(prev => prev + 1)

Explain:

  • 👉 setState-kulla function kudukrom
  • 👉 athula previous value varuthu

✅ 5. Real Example (Button)

<button onClick={() => setCount(prev => prev + 1)}>
Increment
</button>

Explain flow:

  • 1️⃣ button click
  • 2️⃣ prev value varum
  • 3️⃣ +1
  • 4️⃣ re-render

✅ 6. Interview / Student Question Answer

Q: Why not use count+1?

Answer:

  • 👉 React async
  • 👉 batching nadakum
  • 👉 so previous state safer
