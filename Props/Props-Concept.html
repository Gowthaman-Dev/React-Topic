1️.Introduction (5 mins);

“React-la oru component-la irukka data, innoru component-ku eppadi anuppuveenga?”

👉 Props introduce pannunga

Props = Properties

Used to pass data from Parent → Child

Parent component-la irunthu Child component-ku data anuppurathukku props use pannrom.

2.Why we use Props?

Situation Explain;

  • *Oru component-la data irukku
  • *UI-la vera component-la display panna venum

👉 Then we use Props;

Board-la points;

  • *Data sharing
  • *Dynamic UI
  • *Reusable components
🧠 Memory Line
* “Props → Pass data
* State → Manage data”

3️.Props Flow Concept ?

Important Rule 🔑
*Props enable unidirectional data flow
*Parent → Child

4️.Props are Immutable

  • *Props are immutable
  • *Child cannot change props

=>Child component-la props-a modify panna mudiyathu.

Parent kudutha data = rule

Child = just reader

5️.Props Drilling

Start with a Problem Story ;

“Oru data App-la irukku
Aana athu GrandChild-ku mattum venum
Middle components-ku thevai illa” 

App
 └── Parent
   └── Child
      └── GrandChild
  
<App userName="Gowtham" />
<Parent userName={userName} />
<Child userName={userName} />
<GrandChild userName={userName} />
  • *Parent & Child use panna maattanga
  • *Just forward pannranga 😵
  • *This is Props Drilling

6️.Problems with Props Drilling

=>Hard to maintain
=>Hard to debug
=>Code messy
=>Components tightly coupled

*Thevai illaatha components moolama data carry pannrathu problem.

7.Solution – useContext

Without useContext With useContext
Props passing Direct access
Confusing code Clean code
Hard to scale Easy to scale

*useContext use pannina data-va global-aa store panni, thevai irukkura component direct-aa access pannalaam.

9.Props vs useContext vs Others;

Purpose Best Tool
UI data Props
App-level shared data useContext
Complex global state Redux / Zustand
Logic reuse Custom Hooks

10. Golden Rules (Final 2 mins 🔑)

🟢 Use Props when

  • *Parent & Child close-aa irukku
  • *UI related data
  • *Reusable components

🟣 Use useContext when

  • *Same data many components-ku venum
  • *App-level data
  • *Props drilling avoid panna

12.Passing Function as Props;

Concept:

👉 Props-la data mattum illa, function-um pass panna mudiyum.

13.Destructuring Props (Clean Code Concept)

Without destructuring:

props.userName
props.age

With destructuring:

const { userName, age } = props;

*Destructuring makes props usage cleaner and more readable.

14.Props vs State (1 slide / 2 mins recap)

Props State
Parent provides Component owns
Read-only Mutable
Cannot change Can change
Used for data flow Used for data control
🧠 Memory:
*Props → Outside data
*State → Inside data

15.What is defaultProps?

Definition:

👉 Props value parent-la irunthu varalana,

👉 default-aa oru value set panna use pannrathu = defaultProps

Why we use defaultProps?

  • *App crash aaga koodaadhu
  • *UI blank-aa irukka koodaadhu
  • *Avoid undefined errors
  • *Better user experience
