```html useContext – Teaching Notes

1️.Problem introduce pannu

“Props-ah ellaa level-kum anuppinaa enna problem varum?”
🧠 Concept:
  • ❌ Props Drilling
  • 👉 Parent → Child → SubChild → DeepChild

Define pannunga:

Props drilling means passing props through multiple component levels even when intermediate components don’t need the data.

2.useContext yen thevai?

  • *Same data
  • *Many components use pannum
  • *Props drilling avoid pannanum

=>useContext use panninaa data-ah ellaa component-kum direct-aa share pannalaam.

3.Context API overview;

  1. createContext
  2. Provider
  3. Consumer (useContext)
Term Meaning
createContext Context create pannuradhu
Provider Data supply pannuradhu
useContext Data receive pannuradhu

Provider illama useContext work aagathu ❌

4. 👉 Flow:

Context create
↓
Provider wrap
↓
useContext consume

Class-la indha question kekkalaam:

  • “Provider yaaru?”
  • “Consumer yaaru?”

5.Example scenario ;

Situation explain pannu:

“Login panna user name ellaa page-layum thevai”

Components:

  • App
  • Navbar
  • Profile
  • Dashboard
❌ Props drilling panna: App → Navbar → Profile
✅ useContext panna: Direct-aa use pannalaam

6.Key rules & points

🔑 Rules:
  • 1.useContext is a hook 👉 Only function component-la use pannalaam
  • 2.Provider must wrap parent component
  • 3.Context value change aana 👉 All consuming components re-render aagum
  • 4.Small & medium data-ku best 👉 Huge frequent update-ku Redux better

7.Compare with Props ;

Props useContext
Parent → Child Global
Props drilling problem No drilling
Simple Shared state

8.Interview questions ;

Ask students 👇

  1. What problem does useContext solve?
  2. What are the parts of Context API?
  3. Provider illaama useContext use panna mudiyuma?
  4. Props vs Context?
  5. When should we NOT use useContext?

9.Teaching order;

  • 5 min – Props drilling problem
  • 5 min – Why useContext
  • 5 min – Context API parts
  • 5 min – Flow explanation
  • 5 min – Real example
  • 3 min – Rules
  • 2 min – Interview Q&A
```
