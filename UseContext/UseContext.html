1️.Problem introduce pannu

“Props-ah ellaa level-kum anuppinaa enna problem varum?”

🧠 Concept:

  • ❌ Props Drilling
  • 👉 Parent → Child → SubChild → DeepChild

Define pannunga:

Props drilling means passing props through multiple component levels even when intermediate components don’t need the data.

2.useContext yen thevai?

  • *Same data
  • *Many components use pannum
  • *Props drilling avoid pannanum

=>useContext use panninaa data-ah ellaa component-kum direct-aa share pannalaam.

3.Context API overview;

  • 1. createContext
  • 2. Provider
  • 3. Consumer (useContext)
Term Meaning
createContext Context create pannuradhu
Provider Data supply pannuradhu
useContext Data receive pannuradhu

Provider illama useContext work aagathu ❌

  • ❌ Props drilling panna: App → Navbar → Profile

6.Key rules & points

🔑 Rules:

  • 1.useContext is a hook 👉 Only function component-la use pannalaam
  • 2.Provider must wrap parent component
  • 3.Context value change aana 👉 All consuming components re-render aagum
  • 4.Small & medium data-ku best 👉 Huge frequent update-ku Redux better

useContext – CODE FLOW

Create → Provide → Consume

🔹 STEP 1: Context create pannuradhu

📁 Usually context folder create pannuvaanga
📄 UserContext.js

import { createContext } from "react"; const UserContext = createContext(); export default UserContext;

🧠 Explain to students:

  • *createContext() → new context object
  • *Idhu data store maadhiri

STEP 2: Provider use pannuradhu (DATA SUPPLY)

📍 Usually App.jsx / App.js

import UserContext from "./UserContext"; import Navbar from "./Navbar"; function App() { const userName = "Rocky"; return ( <UserContext.Provider value={userName}> <Navbar /> </UserContext.Provider> ); } export default App;

🧠 Explain clearly:

  • *Provider → data supply pannum
  • *value prop-la data pass pannrom
  • *Navbar + inside irukka ellaa components-kum access irukkum

📌 Important warning ⚠️:

Provider illaama useContext work aagathu ❌

🔹 STEP 3: useContext use pannuradhu (DATA RECEIVE)

📄 Navbar.jsx

import { useContext } from "react"; import UserContext from "./UserContext"; function Navbar() { const user = useContext(UserContext); return ( <h1>Welcome {user}</h1> ); } export default Navbar;

🧠 Explain:

  • *useContext(UserContext) → data edukkum
  • *Props use pannala
  • *Direct access ✅

📌 One-line explanation:
useContext is used to consume data from context.

FULL FLOW REVISION (CLASS-LA SOLLRA DIALOGUE)

  • createContext la context create pannom
  • Provider la data supply pannom
  • useContext la data receive pannom

🔥 Simple 🔥

🔷 useContext with OBJECT (REAL WORLD)

Provider:

const user = { name: "Rocky", role: "Admin" }; <UserContext.Provider value={user}>

Consumer:

const user = useContext(UserContext); <p>{user.name}</p> <p>{user.role}</p>

🧠 Mention pannunga:

  • *value string / object / array / function anything irukkalaam
