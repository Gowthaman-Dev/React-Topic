🔥 Step 1 – First Question to Students

👉 “Component render aana udane oru vela nadakka vendum na React-la epdi panrom?”

Examples sollu:

  • ✅ API call
  • ✅ Page load aagumbodhu data show panna
  • ✅ Title change panna
  • ✅ Login user info eduka
Apram sollu:
👉 Itha panna use pannrathu than = useEffect

✅ Definition (Tamil + English)

useEffect na enna?
👉 Component render aanathuku apram run aagura function.
Tamil:
Component screen-la varumbodhu illa state change aagumbodhu automatic-aa oru code run panna use pannrathu = useEffect
English (one line):
👉 useEffect runs side effects after rendering.

🧠 Real Life Example

Fan switch podra madhiri:
Room kulla pona udane fan ON
  • Room = Component
  • Fan = useEffect

✅ Basic Syntax

useEffect(() => {
  console.log("Component Loaded");
}, []);
Explain:
  • 🔹 First argument → function
  • 🔹 Second argument → dependency array

🔥 CORE CONCEPTS (IMPORTANT)

Tomorrow class-la ithu MUST:

1️⃣ Empty Dependency []

useEffect(()=>{
 console.log("Only once");
},[])
Meaning:
👉 Component first time load aagumbodhu mattum run aagum.
Tamil:
Page open pannumbodhu oru thadava than nadakkum.
(API calls)

2️⃣ With Dependency

useEffect(()=>{
 console.log(count);
},[count])
Meaning:
👉 count change aana every time run aagum.
Tamil:
Count maarina udane effect run aagum.

3️⃣ Without Dependency

useEffect(()=>{
 console.log("Always");
})
Meaning:
👉 Every render-kum run aagum.
Tamil:
Button click pannalum, state update aanaalum – yella neramum odum.

⚠️ Interview Question

useEffect types?
  • 1️⃣ [] → one time
  • 2️⃣ [state] → state change
  • 3️⃣ nothing → every render

✅ 1️⃣ useEffect runs AFTER render (Very Important Concept)

Students usually ninaippanga:
👉 useEffect = render kulla nadakkum ❌
Clarify pannunga:
Truth:
👉 First component render aagum
👉 Apram than useEffect run aagum

Example sollu:

console.log("Render");

useEffect(()=>{
 console.log("Effect");
},[])
Output: 
Render
Effect
Tamil:
Screen first draw aagum → apram effect
Interview question varum 👇
👉 useEffect before render ah after render ah?
Answer:
🔥 After render

✅ 2️⃣ Dependency Array = Watch List concept

Explain like CCTV 😄 
useEffect(()=>{
 ...
},[count,name])
Tamil:
👉 count or name change aana mattum effect odum.
Explain sentence:
Dependency array na – entha variable change aana effect run aaganum nu sollra list

✅ 3️⃣ Infinite Loop Problem (Very common mistake)

Show mistake: 
useEffect(()=>{
 setCount(count+1)
})
Explain:
  • ❌ dependency kudukkala
  • ❌ state update panrom
Result:
👉 Infinite loop 🔁
Tamil:
Effect → setState → render → effect → setState → render 😵
Students-ku romba useful warning.

✅ 4️⃣ Multiple useEffect use panna mudiyum

Most beginners ninaippanga:
👉 Oru component-la oru useEffect than ❌
Teach: 
useEffect(()=>{},[])
useEffect(()=>{},[count])
Tamil:
Different works-ku different useEffect use pannalam
