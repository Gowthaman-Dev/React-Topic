🧠 1️⃣ First – Problem create pannunga (5 mins)

Students-kitta first kelunga:

👉 “React la value change panninaa enna nadakkum?”

Answer:
✅ Component re-render aagum.

Appuram example kaatungaa:

const [count,setCount] = useState(0)

Button click → count change → component rerender.

Then question:

👉 “Rerender illama value store panna mudiyuma?”

👉 “DOM element ah direct ah access panna mudiyuma?”

Inga thaan useRef entry 🚪

✨ 2️⃣ useRef definition (simple Tamil)

📌 useRef enna?

  • 👉 React hook.
  • 👉 Value store pannum.
  • 👉 Rerender pannaathu.
  • 👉 DOM element direct ah access panna use pannalaam.

One English line:

useRef lets you persist values and access DOM elements without causing re-renders.

🔥 3️⃣ Syntax explain pannunga

const myRef = useRef(initialValue)

Explain:

  • myRef = object
  • myRef.current = actual value

4️⃣ Concept – useState vs useRef (IMPORTANT)

useState useRef
rerender rerender illa
UI update UI update illa
state reference

Tamil:

👉 useState = screen change

👉 useRef = background memory

🎯 5️⃣ Example 1 – Counter without rerender (10 mins)

import { useRef } from "react" function App() { const countRef = useRef(0) const handleClick = () => { countRef.current += 1 console.log(countRef.current) } return ( <> <button onClick={handleClick}>Click</button> ) }

Explain:

✔ click panna console la increase

❌ screen change illa

Say this clearly:

🔥 useRef update panninaa component rerender aagaathu

🎯 6️⃣ Example 2 – Input focus (real-world)

const inputRef = useRef() <button onClick={()=>inputRef.current.focus()}> Focus </button> <input ref={inputRef}/>

Explain:

👉 ref attribute DOM ah connect pannum

👉 .focus() direct browser method

Students will like this 😄

🧩 7️⃣ When to use useRef? (Interview point)

Tell them:

  • ✅ Input focus
  • ✅ Previous value store
  • ✅ Timer / interval id
  • ✅ Rerender vendam value store panna

⚠️ 8️⃣ Common mistakes

  • useRef() outside component ❌
  • .current maranthudrathu
  • UI update expect pannrathu

✅ 1️⃣ Previous value track pannradhu (VERY IMPORTANT)

Most people skip this — interview-la kekka chance iruku.

import { useEffect, useRef, useState } from "react" function App() { const [count, setCount] = useState(0) const prevCount = useRef() useEffect(() => { prevCount.current = count }, [count]) return ( <> <h1>Current: {count}</h1> <h2>Previous: {prevCount.current}</h2> <button onClick={()=>setCount(count+1)}>+</button> ) }

Explain like this:

👉 useState = current value

👉 useRef = previous value memory

Say this sentence:

🔥 useRef component rerender aagum pothum value keep pannum

✅ 2️⃣ Real reason why useRef exists

Students kitta clear ah sollunga:

React la 2 type memory iruku:

🟥 State memory → rerender

🟩 Ref memory → no rerender

Tamil line:

👉 Screen change venum na → useState

👉 Screen change vendam na → useRef

🧠 Simple teaching analogy (kids-ku super)

Tell this:

useRef = pocket notebook

(update pannalam, board change illa)
